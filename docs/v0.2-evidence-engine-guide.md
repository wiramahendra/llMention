# LLMention v0.2 Evidence Engine - User Guide

## Overview

LLMention v0.2 introduces an evidence-first approach to Generative Engine Optimization (GEO). This new workflow provides:

- **Project-level configuration** via `llmention.toml`
- **Multi-sample audits** for statistical significance
- **Prompt discovery** with intent categorization
- **Raw evidence storage** for transparency
- **Content generation** from audit gaps
- **Before/after comparison** of audit runs

## Quick Start (Evidence Engine)

```bash
# 1. Initialize a new project
llmention init

# 2. Discover prompts based on your project config
llmention prompts discover

# 3. Run a mock audit (no API keys needed)
llmention audit run --models mock --samples 3

# 4. View audit results
llmention audit list
llmention audit show 1

# 5. Generate a markdown report
llmention report --output ./reports/

# 6. Generate content assets from gaps
llmention generate --output ./generated/

# 7. Compare two audit runs
llmention audit compare --before 1 --after 2
```

## Commands Reference

### `llmention init` - Initialize Project

Creates a `llmention.toml` file in the current directory.

```bash
# Interactive mode
llmention init

# Non-interactive with defaults
llmention init --yes

# Specify project details
llmention init --name "MyProject" --website "https://example.com" --category "developer tool"

# Overwrite existing config
llmention init --force
```

**Generated `llmention.toml`:**
```toml
[project]
name = "MyProject"
website = "https://example.com"
category = "developer tool"
description = ""
audience = []

[competitors]
names = []

[keywords]
topics = []

[providers]
default = "ollama"
models = ["ollama:llama3.2"]

[audit]
samples_per_prompt = 3
temperature = 0.2
store_raw_responses = true
```

### `llmention prompts` - Prompt Management

#### `prompts discover`

Generates prompts based on your project configuration.

```bash
# Discover up to 50 prompts (default)
llmention prompts discover

# Limit to 20 prompts
llmention prompts discover --limit 20
```

**Prompt Categories:**
- **Category**: "Best tools for X"
- **CompetitorAlternative**: "Alternatives to CompetitorY"
- **ProblemAware**: "How to check if..."
- **SolutionAware**: "What is ProjectX"
- **Comparison**: "Compare X vs Y"
- **BuyerIntent**: "Which should I use"

#### `prompts list`

Shows stored prompts with metadata.

```bash
llmention prompts list
```

### `llmention audit` - Evidence-First Audits

#### `audit run`

Runs a multi-sample audit across configured models.

```bash
# Run with default settings
llmention audit run

# Use mock provider (no API keys needed)
llmention audit run --models mock --samples 3

# Override specific settings
llmention audit run --samples 5 --temperature 0.5 --models ollama:llama3.2

# JSON output
llmention audit run --json
```

**Metrics Collected:**
- Mention rate: % of responses mentioning your project
- Citation rate: % of responses with URLs to your project
- Recommendation rate: % actively recommending your project
- Position: Where in the response you're mentioned
- Sentiment: Positive, neutral, or negative

#### `audit list`

Shows previous audit runs.

```bash
# List last 20 runs
llmention audit list

# List last 10 runs
llmention audit list --limit 10
```

#### `audit show`

Shows details of a specific audit run.

```bash
llmention audit show 42
```

### `llmention report` - Generate Reports

Creates markdown evidence reports from audit results.

```bash
# Generate report from latest audit
llmention report

# Generate report from specific run
llmention report --run 42

# Specify output directory
llmention report --output ./reports/

# Include full raw responses
llmention report --full

# Force overwrite existing report
llmention report --force
```

**Report Sections:**
- Executive Summary
- Detailed Metrics
- Results by Model/Provider
- Prompt-Level Results
- Competitor Analysis
- Citations Found
- Content Gaps & Recommendations
- Raw Evidence Appendix (with --full)
- Methodology & Caveats

### `llmention generate` - Content Generation

Generates markdown assets based on audit gaps.

```bash
# Generate from latest audit
llmention generate

# Generate from specific audit run
llmention generate --from-audit 42

# Specify output directory
llmention generate --output ./content/

# Force overwrite existing files
llmention generate --force
```

**Generated Assets:**
- Comparison pages
- Alternatives pages
- Use case pages
- FAQ pages
- Docs summaries
- README patches
- llms.txt

### `llmention audit compare` - Compare Audit Runs

Shows before/after visibility movement.

```bash
# Compare two runs (terminal output)
llmention audit compare --before 10 --after 20

# JSON output
llmention audit compare --before 10 --after 20 --format json
```

**Comparison Metrics:**
- Visibility score delta
- Mention rate delta
- Citation rate delta
- Recommendation rate delta

### `llmention diagnose` - URL Diagnostics

Checks basic crawlability and AI readiness.

```bash
llmention diagnose https://example.com
```

**Checks:**
- Homepage reachability
- robots.txt presence
- sitemap.xml presence
- llms.txt presence

## Complete Workflow Example

```bash
# 1. Create a new project directory
mkdir my-geo-project && cd my-geo-project

# 2. Initialize LLMention
llmention init --name "MyAwesomeTool" --website "https://myawesometool.dev" --category "CLI tool"

# 3. Edit llmention.toml to add competitors and keywords
# (Use your favorite editor)

# 4. Discover prompts
llmention prompts discover

# 5. Review discovered prompts
llmention prompts list

# 6. Run first audit with mock provider
llmention audit run --models mock --samples 3

# 7. Check the results
llmention audit list
llmention audit show 1

# 8. Generate a report
llmention report --output ./reports/

# 9. Generate content from gaps
llmention generate --output ./content/

# 10. Review generated content and publish it
# ... (edit and publish the generated files)

# 11. Run a second audit after publishing
llmention audit run --models mock --samples 3

# 12. Compare the two runs
llmention audit compare --before 1 --after 2
```

## Configuration

### Project Config (`llmention.toml`)

```toml
[project]
name = "MyProject"
website = "https://myproject.dev"
category = "AI visibility tool"
description = "A local-first AI visibility workbench for developers"
audience = ["indie hackers", "solo founders", "open-source maintainers"]

[competitors]
names = ["CompetitorA", "CompetitorB"]

[keywords]
topics = ["AI visibility", "GEO", "ChatGPT mentions"]

[providers]
default = "ollama"
models = [
    "ollama:llama3.2",
    "openai:gpt-4o-mini",
    "anthropic:claude-3-haiku"
]

[audit]
samples_per_prompt = 3
temperature = 0.2
store_raw_responses = true
```

### Global Config (`~/.llmention/config.toml`)

API keys and global settings remain in the global config:

```toml
[providers.openai]
api_key = "sk-..."
model = "gpt-4o-mini"
enabled = true

[providers.ollama]
base_url = "http://localhost:11434"
model = "llama3.2"
enabled = true
```

## Data Storage

All evidence is stored locally in SQLite:

- `~/.llmention/evidence.db` - Audit runs, results, prompts
- `~/.llmention/mentions.db` - Legacy data (preserved)

## Privacy

- All project data stays in `llmention.toml` (local)
- All audit data stays in local SQLite
- Only prompts sent to configured providers
- No telemetry or external data transmission
- API keys never logged or stored in project files

## Mock Provider

For testing without API keys:

```bash
llmention audit run --models mock --samples 3
```

The mock provider returns predictable responses for testing the workflow.

## Tips

1. **Start with mock**: Test your workflow with `--models mock` before using real providers
2. **Use samples**: Set `samples_per_prompt = 3` or higher for statistical significance
3. **Iterate**: Run audits, generate content, publish, then re-audit to measure improvement
4. **Compare**: Use `llmention compare` to track progress over time
5. **Store raw responses**: Keep `store_raw_responses = true` for debugging and evidence

## Troubleshooting

### "No llmention.toml found"
Run `llmention init` first to create a project configuration.

### "No providers configured"
Either:
- Use `--models mock` for testing
- Configure providers in `~/.llmention/config.toml`

### "No prompts found"
Run `llmention prompts discover` to generate prompts from your project config.

### "No audit runs found"
Run `llmention audit run` first to create audit data.

## Migration Notes

### From v0.1 (Legacy Commands)

The v0.2 evidence engine is fully backward compatible:
- Existing commands continue to work (now named `audit-legacy`, `report-legacy`, `generate-legacy`)
- Legacy data in `mentions.db` is preserved
- New evidence data goes to `evidence.db`

### From v0.2 Temporary Commands

The temporary `*2` command names have been promoted to clean command names:

| Old Command | New Command |
|-------------|-------------|
| `llmention init2` | `llmention init` |
| `llmention prompts2 discover` | `llmention prompts discover` |
| `llmention prompts2 list` | `llmention prompts list` |
| `llmention audit2 run` | `llmention audit run` |
| `llmention audit2 list` | `llmention audit list` |
| `llmention audit2 show` | `llmention audit show` |
| `llmention report2` | `llmention report` |
| `llmention generate2` | `llmention generate` |
| `llmention diagnose2` | `llmention diagnose` |
| `llmention compare` | `llmention audit compare` |

The old `*2` commands will continue to work with a deprecation warning.
